Manual Técnico: Transacciones de Préstamos

Módulo: Flujo de Material Bibliográfico | Versión: 1.0

1. Descripción del Proceso

Este módulo gestiona el cambio de estados físicos de los libros. Enlaza un Lector (DNI) con un Ejemplar (ISBN) generando un contrato temporal. Trabaja de la mano con el motor de base de datos para prevenir duplicidades.

2. Procedimiento de Préstamo (Salida)

  • Paso 1: Digitar el DNI del Lector. El sistema debe confirmar que no posee multas pendientes.
  • Paso 2: Digitar el ISBN o utilizar el lector de código de barras.
  • Paso 3: Verificar el estado en pantalla. Solo se permite la transacción si el estado es 'Disponible'.
  • Paso 4: Seleccionar en el calendario la Fecha Límite de Devolución. (La fecha actual se asigna automáticamente como inicio).
  • Paso 5: Confirmar préstamo. El inventario actualizará internamente el libro a 'No Disponible'.

3. Procedimiento de Devolución (Ingreso)

  • Busque el préstamo activo ingresando el DNI del usuario en la barra de búsqueda de la tabla inferior.
  • Seleccione el registro y presione el botón 'Finalizar Préstamo'.
  • El sistema leerá la fecha actual y la comparará con la fecha límite. Si hay mora, aplicará los cálculos definidos en la Configuración Global.
  • El libro retorna inmediatamente a su estado 'Disponible' para el siguiente usuario.

4. Restricciones Críticas

  • No intente prestar libros digitales (URL): Los recursos digitales no tienen control de inventario temporal, son de acceso permanente.
  • No modifique fechas de transacciones pasadas: Por integridad de base de datos, las fechas de préstamos finalizados son inmutables.

🛠️ FAQ y Troubleshooting

Problema: "El sistema bloquea el préstamo de un libro físico".
Causa: El libro probablemente no ha sido retornado por el usuario anterior. Verifique su estado en el módulo de 'Gestión de Libros'.
